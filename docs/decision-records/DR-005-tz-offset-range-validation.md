# DR-005: TzOffset の範囲バリデーションと pub(all) 維持

- **日付**: 2026-03-16
- **ステータス**: Accepted

## 決定

### 1. parse_tz_offset に ±24 時間の範囲チェックを追加

`parse_tz_offset` でパースした結果が ±1440 分（±24 時間）を超える場合は `ParseError` を返す。

### 2. TzOffset は `pub(all)` を維持

`Hour(0)` と `Utc` の重複は許容する。スマートコンストラクタによる正規化は行わない。

## 理由

### 範囲チェック

- TzOffset は「タイムゾーンオフセット」を表す型であり、任意の時間シフトではない
- 任意の時間シフトが必要なら `Duration` + `EpochTime::add_duration` を使うべき
- `+99` のような入力はほぼ確実にタイポであり、エラーで検出する価値がある
- ±24 時間は RFC 3339 の範囲（±23:59）をわずかに超える程度で、実世界の全 TZ（UTC-12 〜 UTC+14）を十分にカバー

### pub(all) 維持

- ユーザーが `Hour(9)` や `Min(330)` を直接構築するのが最もシンプルな API
- `make_parse_datetime(default_tz_offset=Hour(9))` のように値を渡す用途が主
- `pub enum`（構築不可・match 可）にすればスマートコンストラクタで `Hour(0)` → `Utc` の正規化が可能だが、利便性とのトレードオフで不採用
- `Hour(0) != Utc` の問題は計算上は無害（同じオフセット 0 分として処理される）
